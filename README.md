# デジタルバンク ランディングページ（カンプ書き起こし実績）

実務レベルのデザインカンプおよび要求仕様をもとに、フィンテック（金融系）サービスをイメージしたレスポンシブWebサイトの実装実績です。  
画面幅に応じて滑らかに変化する「流体レスポンシブ（SCSS）」や、画面幅（ブレイクポイント）の切り替えに伴うイベントリスナーの追加・削除など、実務を想定した丁寧なコンポーネント実装を行いました。

## 概要

### 要件・機能

支給されたデザインカンプ（モバイル：375px / デスクトップ：1440px）から正確にスタイルを起こし、以下のインタラクションを実装しています。

- **流体レスポンシブレイアウト：**  
ブレイクポイント（768px / 1440px）での唐突なレイアウト崩れを防ぎ、スマートフォン・タブレット・PCのあらゆる画面幅においてデザインの比率を維持したまま縮尺が可変（Fluid）するビジュアルを構築。
- **ブレイクポイント連動型ハンバーガーメニュー：**  
モバイル環境（1439px以下）のみで機能するナビゲーション。展開時は背景を暗転させるオーバーレイマスクが同期し、背後のコンテンツへの干渉を防ぐ設計。
- **交差監視によるフェードイン：**  
`Intersection Observer API`（交差監視API）を利用した、スクロールに連動するフェードイン演出。

## 開発プロセス

### ディレクトリ構造

Vite環境における静的アセット管理およびSassのコンパイル構造に基づき、関心の分離（Separation of Concerns）を意識したクリーンなディレクトリ設計を行っています。ローカルの `node_modules/` およびビルド成果物（`dist/`）は `.gitignore` を用いてGit管理対象から除外しています。

```text
.
├── .github/
│   └── workflows/           # GitHub Actions自動デプロイ設定
├── .gitignore               # Git管理除外設定（node_modules、distの除外）
├── README.md                # 本ドキュメント
├── index.html               # メインHTML（エントリーポイント）
├── package-lock.json        # 依存ライブラリのバージョンロック
├── package.json             # プロジェクト依存関係・ビルドスクリプト定義
├── preview.jpg              # README表示用のプレビュー画像
├── vite.config.js           # Vite設定ファイル
├── design/                  # 支給されたデザインカンプ（Git管理）
│   ├── active-states.jpg
│   ├── desktop-design.jpg
│   ├── mobile-design.jpg
│   └── mobile-navigation.jpg
├── public/                  # 成果物にそのままコピーされる静的アセット
│   └── images/              # ロゴ、各種アイコン、背景用SVG/画像等
└── src/                     # ソースコード開発ディレクトリ
    ├── main.js              # 全体統合用のメインJS
    ├── css/
    │   └── style.scss       # 変数・コンポーネントを包括するSCSS
    └── js/
        └── script.js        # DOM操作・ナビゲーション制御を行うVanilla JS
```
### 使用技術・環境
- 言語・マークアップ: HTML5 / SCSS (Sass) / Vanilla JS (ES6+)
- リセットCSS: destyle.css (npm管理・SCSS側でのインポート)
- ビルドツール: Vite
- 環境自動化: GitHub ActionsによるGitHub Pagesへの自動CI/CDパイプライン構築

### こだわった技術的アプローチ
**1. セマンティック・マークアップと適切なHTML構造**

セマティクスを意識し、header, main, section, footer などの適切な要素を選定。セクションごとに適切な h1〜h3 の階層構造を意識し、SEOおよび支援技術（スクリーンリーダー）に配慮した論理的な構造でマークアップを行いました。  
また、フッター部の各種SNSリンクには無駄のないインライン・リスト構造を採用し、要素間のコンポーネントの独立性を高めています。

**2. ビューポートに追従する流体レスポンシブ（SCSS）**

font-size、padding、margin、要素の横幅にいたるまで、ビューポート幅を基準とした vw 単位をベースにスタイルを設計。  
これにより、特定のブレイクポイント間で文字が溢れたり、意図しないレイアウト崩れを起こしたりすることなく、すべてのデバイスでカンプ通りのトーン＆マナーを維持できるようにしています。

**3. ブレイクポイントを跨ぐイベントハンドリング（Vanilla JS）**

モバイルナビゲーションの実装において、デスクトップ表示（1440px以上）に切り替わった際には不要となるイベントリスナー（click）を removeEventListener で動的に破棄するロジックを構築しました。  
PC環境においてバックグラウンドで無駄なリスナーが走り続けるのを防ぐ設計にしています。

**4. 交差監視APIによるスクロール連動演出（Vanilla JS）**

従来の window.addEventListener('scroll') を用いた過剰なスクロールイベント発火によるブラウザ負荷を避けるため、IntersectionObserver を採用。  
要素が画面内に10%交差した瞬間（threshold: 0.1）に .is-active クラスを付与してアニメーションを発火させ、同時に observer.unobserve() によって即座に監視を終了。無駄なイベント発火を抑え、滑らかなスクロールを実現しました。

### 今回の開発での気づき（今後挑戦したいこと）
**アクセシビリティ（WAI-ARIA）の考慮：**    
現状のハンバーガーメニュー開閉に伴い、スクリーンリーダー利用者にもナビゲーションの状態が明示的に伝わるよう、aria-expanded="true/false" の動的制御やキーボード操作時のフォーカス制御の実装を追加していきたいと考えています。

**JavaScriptのモジュール化：**  
現在は script.js 内にすべてのUI制御を記述していますが、今後の拡張性やメンテナンス性を考慮し、各UIロジックを独立したESモジュールとして分割管理する構成へのリファクタリングに挑戦したいです。
